# 快速开始 - 网页智能对话

## 安装依赖

```bash
# 前端依赖
bun install

# Rust 依赖会在构建时自动安装
```

## 启动开发服务器

```bash
bun run tauri dev
```

## 使用步骤

### 1. 配置 LLM

首次使用前，需要在"设置"页面配置：

- **Base URL**: 你的 LLM API 地址（例如：`https://api.longcat.chat/openai/v1`）
- **API Key**: 你的 API 密钥
- **Models**: 可用的模型列表

### 2. 添加网页

1. 点击侧边栏导航到"网页智能对话"
2. 点击"添加网页"按钮
3. 输入要解析的网页 URL（例如：`https://www.npmjs.com/package/turndown`）
4. 等待解析完成（状态会显示为"成功"）

### 3. 开始对话

1. 在顶部选择要使用的 AI 模型
2. 在输入框中输入问题（例如："这个网页主要讲了什么？"）
3. 按 `Ctrl + Enter` 或点击"发送"按钮
4. AI 会基于解析的网页内容回答你的问题

### 4. 多轮对话

- 继续输入问题，AI 会记住之前的对话历史
- 可以随时切换模型继续对话
- 点击"清空对话"可以重新开始

### 5. 消息管理 ✨

**右键菜单功能：**
- 在任意消息上右键点击
- 选择"复制"可以复制消息内容到剪贴板
- 选择"删除"可以删除该条消息

**Markdown 渲染：**
- AI 的回复会自动渲染为 Markdown 格式
- 支持代码块语法高亮
- 支持表格、列表、引用等丰富格式

### 6. 快捷工具

点击顶部的"HTML转Markdown"按钮，可以：

- **HTML 输入**：直接粘贴 HTML 代码进行转换
- **URL 输入**：输入网页 URL 自动获取并转换
- 转换完成后可以一键复制结果

## 常见问题

### Q: 网页解析失败怎么办？

A: 可能的原因：
- 网络连接问题
- 目标网站有反爬虫机制
- URL 格式不正确
- 网站需要登录才能访问

### Q: 为什么 AI 回答不准确？

A: 建议：
- 确保网页内容已成功解析
- 尝试更具体的问题
- 切换到更强大的模型
- 清空对话重新开始

### Q: 可以同时解析多少个网页？

A: 理论上没有限制，但建议：
- 不超过 5 个网页
- 注意总字符数不要超过模型的上下文限制
- 过多内容可能影响 AI 回答质量

### Q: 对话历史会保存吗？

A: 当前版本对话历史只保存在内存中，刷新页面会清空。未来版本会添加持久化功能。

## 技术说明

### 为什么使用 Rust 处理网页抓取？

1. **绕过 CORS**：浏览器的跨域限制不适用于 Rust 后端
2. **自定义请求头**：可以设置 User-Agent 等浏览器禁止的头部
3. **更好的性能**：Rust 的异步 HTTP 客户端高效稳定
4. **统一错误处理**：在后端统一处理网络和解析错误

### 数据流程

```
用户输入 URL
    ↓
前端调用 Tauri 命令
    ↓
Rust 后端使用 reqwest 获取网页
    ↓
使用 scraper 清理 HTML
    ↓
使用 html2md 转换为 Markdown
    ↓
返回给前端显示
    ↓
用户提问时，Markdown 作为上下文传给 LLM
```
